Behaviour & data notes for IT developer

  • All three modals open as a bottom-sheet on mobile, centred dialog on desktop (max-width 480px).
  • Glucose reading type pill (FBS / 2hPP / Random) is required before Save is enabled.
  • Units default to mmol/L (MY/SG standard) — stored in mmol/L internally regardless of UI toggle.
  • Date + time auto-fill to now(), and stay editable. Server stores both logged_at and created_at.
  • Target zone bar colour-codes live: green 3.9–7.0 (FBS) / <7.8 (2hPP), amber up to prediabetic, red above.
  • If reading < 3.0 or > 14.0 mmol/L → trigger AI-11 Emergency Escalation immediately on save.
  • Meal photo upload triggers AI-04 (vision analysis) + AI-12 (feedback draft) in parallel.
  • Weight / waist saves to vitals_log (AI-02 data table) with delta computed against the last entry.
  • Context tags feed AI-06 (symptom extraction) so "feeling tired" / "shaky" can flag a hypo trend.
  • Every successful save shows a green toast at top of the Home screen ("Reading saved · 已保存") and refreshes the Home chart + KPI tiles.
